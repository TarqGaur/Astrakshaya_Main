Comment Fetcher AI - Complete Technical Documentation

Complete technical architecture of Echo's zero-backend analytics system. From comment ingestion via APIs to AI-powered sentiment analysis and real-time visualization, every component works seamlessly in the browser without server infrastructure.

1. Project Overview

Comment Fetcher AI is a zero-backend, single-HTML-file social media analytics suite. It takes a YouTube or Instagram post URL, scrapes all comments via third-party APIs, runs multi-dimensional AI analysis in ONE API call, and renders 10+ visual sections from the user's browser.

No Server Required

No backend infrastructure. No database. No framework overhead. Every operation runs directly in your browser using public APIs and the user's own API keys.

7. Apify Actor Polling Pattern

async function waitApifyRun(runId) { for (let i = 0; i < 20; i++) { await sleep(5000); // Wait 5 seconds const status = await fetch(`/v2/actor-runs/${runId}`) .then(r => r.json()) .then(s => s.data.status); if (status === 'SUCCEEDED') { return datasetId; // Success! } if (['FAILED','ABORTED','TIMED-OUT'].includes(status)) { throw Error(`Actor failed: ${status}`); } } throw Error('Actor polling timeout'); } // Polls every 5s, max 100 seconds total wait // Returns datasetId → then GET /datasets/{id}/items

Why Polling?

Apify actors run asynchronously. We poll the status endpoint every 5 seconds until completion. Maximum 20 attempts equals 100 second timeout. If the actor still hasn't completed, we gracefully fail and show error to user.

8. Unified AI Analysis - The Mega Prompt

Core Architectural Decision

Instead of 9 separate API calls (one per analysis type), all analysis happens in a single API call. This provides massive cost and speed benefits.

Why One Call Instead of Nine?

Old Approach

  • 9 separate sentiment/topic/profile calls
  • 9 calls times 8 seconds equals 72 seconds total
  • Comments sent 9 different times
  • 9 times higher token cost
  • Complex error handling

New Approach

  • 1 unified mega-prompt call
  • 1 call times 15 seconds total
  • Comments sent only once
  • Approximately 80% lower token cost
  • Single error state

Mega Prompt Structure

Input to AI: ├── First 200 comment texts (truncated to 150 chars each) ├── Country hint data (username + 200 chars of text) ├── Full JSON response schema (9 sections) └── Platform context (YouTube/Instagram) Model: openrouter/auto (hunter-alpha) Max tokens: 6000 Temperature: 0.2 (deterministic results) Reasoning: enabled (for explanation)

Unified Response Schema

{ "sentiment": { "counts": {}, "overallMood": "", "insight": "" }, "audience": { "items": [{ rank, icon, category, title, description }] }, "topic": { "headline": "", "tags": [{ label, count }] }, "profiles": { "profiles": [{ name, emoji, description, share% }] }, "country": { "countries": { "India": {pos, neg, neu}, ... } }, "spam": { "spam": {}, "bots": {}, "riskLevel": "LOW" }, "summary": { "sentiment": "", "action": "" }, "personas": { "personas": [{ name, emoji, age, mood, engagement }] }, "prediction": { "like": 78, "dislike": 22, "confidence": "HIGH" } }

9. JSON Parsing Strategy

function parseJSON(raw) { // Strip markdown code blocks that models sometimes add raw = raw.replace(/```json/gi, '').replace(/```/gi, '').trim(); // Extract first {...} JSON block using regex const match = raw.match(/\{[\s\S]*\}/); if (!match) { throw new Error('No JSON found in AI response'); } return JSON.parse(match[0]); } // Result: Handles models that add extra prose before/after JSON

Robust Parsing

Some AI models wrap responses in markdown code blocks or add explanatory text. This regex-based approach extracts the pure JSON object regardless of surrounding text.

12. Country Detection Algorithm

Country is NOT detected by IP (no server) - AI infers from comment content:

Input sent to AI: user:username123 comment text here... user:another_user another comment... AI detects via: ├── Language analysis (Hindi → India) ├── Slang/cultural references (bhai, yaar → India/Pakistan) ├── Username patterns (numbers, scripts, emojis) └── Place/celebrity/event mentions Output: { "India": { "pos": 40, "neg": 10, "neu": 20 }, "Brazil": { "pos": 25, "neg": 15, "neu": 10 }, ... }

Hate Pie Derivation (Zero Extra API Cost)

// Extracted from country data in ONE line: const hateData = Object.entries(countryData) .filter(([, d]) => d.neg > 0) .sort((a, b) => b[1].neg - a[1].neg) .slice(0, 8); // Top 8 hate sources // Pure JavaScript - zero extra API tokens

Cost Efficiency

By asking the AI to return country-level sentiment in the mega prompt, we can derive the Hate Pie chart entirely with client-side JavaScript. No additional API call needed.

16. Error Handling Strategy

Layer Error Handling User Feedback
Apify Polling Max 20 attempts × 5s = 100s timeout, throws on FAILED/ABORTED Red error box in fetcher section
YouTube API Stops at token exhaustion or maxResults reached Partial results with count shown
AI parseJSON Regex extracts JSON block, throws if not found Each section shows: "Analysis failed"
Image Load onerror → swap to gradient placeholder Fallback background still looks good
Prediction Fallback If AI fails, uses hardcoded {like:78, dislike:22} Section still renders with data

Graceful Degradation

If any single analysis fails, other sections still render. If the hero image fails to load, we show a clean gradient. The app never crashes. It always shows something.

17. Performance Optimizations

  • One AI call instead of 9: Approximately 80% cost and time reduction (biggest win)
  • Parallel fetching: Post details scraper and comments fetch run simultaneously
  • Comment sampling: AI only sees first 200 comments (truncated to 150 chars) to avoid context limit
  • Country hints: Username prepended to comment improves accuracy without extra API call
  • No frameworks: Zero bundle size, instant load, no React or Vue overhead
  • Lazy section display: Sections start hidden, revealed only after data arrives
  • requestAnimationFrame: Bar chart animations only trigger after DOM paint (60fps smooth)
  • Cached API tokens: Tokens stored in localStorage, no re-entry after first setup
